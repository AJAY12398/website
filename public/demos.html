This page demonstrates some of the cool things you can do with ProseMirror. There are no docs yet, but you can look at the source code for this page and ProseMirror itself to get some idea what coding with this library looks like.

A fall-back editor for Markdown

## A Markdown document Many people like the directness of writing [Markdown-style](http://commonmark.org) content. It definitely beats BBCode. However, for people who aren't already used to internalizing arcane syntactic rules, the format is *not* very approachable. You could set up your commenting or authoring system to allow people to **choose** between text-based Markdown input and a familiar editor interface built on ProseMirror.

Custom document elements

Imagine a content management situation that revolves around dinosaurs. In this editor, dinosaurs are first-class elements in the document model. They can appear anywhere where text can appear.

For example, this paragraph is full of them.

You can insert dinosaurs by typing their names enclosed by brackets, such as [pterodactyl]. You'll get a simple completion tooltip as you do so, and can press tab to select the top completion. The menu bar also contains a dinosaur-insertion submenu.

Dinosaurs can be selected, copied, pasted, dragged, and so on.

Note: The document-model customization API is not finished yet. It is powerful enough to kludge in something like this, but there's a bunch of work to do before we can write robust extensions that interact properly with all parts of the system.
